Frequently Asked Questions

What is Snares Linux?

Snares Linux is a secure, general-purpose Linux distribution built on top of Void Linux. While it inherits Void's package archive and uses the XBPS package manager, Snares Linux provides its own desktop experience, installer, security defaults, and system configuration.

Snares Linux aims to combine the simplicity and performance of Void Linux with a modern desktop experience and strong security features.

How is Snares Linux different from Void Linux?

Snares Linux is built on Void Linux but provides a different user experience and set of defaults.

  • Guided graphical installer
  • LUKS disk encryption enabled by default
  • dinit as the default init system
  • AppArmor enabled and enforced by default
  • GNOME desktop environment
  • First-boot configuration wizard
  • Curated desktop experience
  • Named releases with rolling package updates

Is Snares Linux based on Void Linux?

Yes. Snares Linux uses Void Linux as its foundation and inherits its package repositories and XBPS package manager.

Most software available for Void Linux can also be installed on Snares Linux.

Which init system does Snares Linux use?

Snares Linux uses dinit as its default init system.

dinit is lightweight, fast, and designed around simple and reliable service management.

Support for OpenRC may be added in the future, but dinit is currently the primary supported init system.

Does Snares Linux use systemd?

No.

Snares Linux does not use systemd and has no plans to adopt it. Instead, it uses dinit as its default init system.

Why does Snares Linux use musl?

Snares Linux is based on the musl variant of Void Linux.

musl is known for its simplicity, small size, security-focused design, and standards compliance.

While most Linux software works correctly on musl, some proprietary applications may only support glibc-based systems.

Which desktop environment does Snares Linux use?

Snares Linux uses GNOME as its official desktop environment.

The desktop experience is designed to be clean, modern, and easy to use while remaining lightweight enough for everyday computing.

Does Snares Linux support Wayland?

Yes.

Snares Linux uses GNOME on Wayland by default. X11 is not a primary target and future development focuses on Wayland support.

What package manager does Snares Linux use?

Snares Linux uses XBPS (X Binary Package System), the same package manager used by Void Linux.

Examples: 


xbps-install firefox
xbps-install -Su

Is disk encryption supported?

Yes.

Snares Linux uses LUKS for full-disk encryption and enables encryption by default during installation.

Users may disable encryption during installation if desired.

What security features are included?

Security is one of the primary goals of Snares Linux.

  • LUKS disk encryption
  • AppArmor enabled by default
  • Hardened kernel configuration
  • Minimal default software installation
  • Regular security updates through XBPS

Is Snares Linux a rolling release?

Snares Linux uses a hybrid release model.

Packages are updated continuously through XBPS, but Snares Linux also publishes named releases that provide new installation media, updated defaults, and major project milestones.

Is Snares Linux ready for daily use?

Not yet.

Snares Linux is currently in active development and should be considered experimental until the first public release is announced.

Who is Snares Linux for?

Snares Linux is designed for users who want a modern desktop Linux distribution with strong security defaults, a straightforward installation process, and the flexibility of the Void Linux ecosystem.

Can I contribute?

Yes.

Contributions are welcome in many forms:

  • Testing development builds
  • Bug reports
  • Documentation improvements
  • Packaging software
  • Code contributions
  • Community support

When will the first release be available?

There is currently no announced release date.

Development is focused on completing the installer, ISO generation system, desktop integration, and first-boot experience before the first public release.
